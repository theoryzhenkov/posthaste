# Branding -- L0

## Name

**PostHaste** (camelCase in wordmark and code references).

From the English expression "post-haste" meaning "with utmost speed" — literally from postal riders writing "haste, post, haste" on urgent letters. The name is:

- Immediately mail-associated ("post")
- Implies speed and efficiency ("haste")
- A real English word with historical depth
- Memorable, distinctive, slightly witty

## Brand energy

Speed, wind, lightness, sky, organic movement. Not heavy industrial — a swift messenger cutting through open air. The aesthetic is "gentle brutalist": information-dense and serious when you need it, friendly and joyful when you don't. A professional power tool that is also a pleasure to use.

Metaphor space: wind and natural force. Not aviation/mechanical, not cute/playful. Elegant, flowing, purposeful speed.

## Audience

Open-source project. Primary audience is power email users (MailMate, mutt, Thunderbird power-user crowd), but the brand should be welcoming to newcomers. Guides, documentation, and onboarding are first-class priorities.

## Logo

Abstract wind/flow mark — curved, organic lines suggesting air current and forward movement. Paired with "PostHaste" wordmark in Geist Sans.

No icon-only mark yet; the wordmark + flow mark work as a unit.

## Typography

- **UI text**: Geist Sans (variable weight). Hierarchy via weight only — regular, semibold, bold. No size variation for hierarchy.
- **Monospace**: Geist Mono for code, raw headers, and technical metadata where needed.
- No custom display font or heading font.

## Information density

- Default: comfortable spacing — readable, efficient, works for everyone.
- User toggle: compact/dense mode for power users who want maximum visible messages.
- Both modes use the same type hierarchy; only row height and padding change.
